Games Without Brains

ü§ñ

Without conditionals, games are boring:

// Always the same outcome
let playerGold = 50;

// No conditional check - always the same result
playerGold = playerGold + 10;
document.getElementById('story').innerHTML =
    "You enter the dungeon and find treasure!";

Problem: No intelligence, no choices matter!

Games with Brains

üß†

With conditionals, games become intelligent:

// Smart game responses using conditionals
let hasTorch = true;
let playerHealth = 100;

if (hasTorch) {
    document.getElementById('story').innerHTML =
        "Your torch lights the way. You find ancient treasure!";
} else {
    document.getElementById('story').innerHTML =
        "It's too dark! You stumble and fall.";
    playerHealth = playerHealth - 10;
}

Result: Player choices and items matter!

The Digital Bouncer

üö™ If statements are like bouncers at a club

They check if you meet the requirements before letting you in

// Basic if statement structure
let playerHealth = 75;

if (playerHealth > 0) {
    document.getElementById('story').innerHTML = "You're still alive!";
}

// If-else for two outcomes
let playerGold = 30;

if (playerGold >= 50) {
    document.getElementById('story').innerHTML = "You can afford the magic sword!";
} else {
    document.getElementById('story').innerHTML = "You need more gold.";
}

Multiple Paths (If-Else Chain)

When you need more than two outcomes:

let playerLevel = 7;

if (playerLevel >= 10) {
    document.getElementById('story').innerHTML = "You're a master adventurer!";
} else if (playerLevel >= 5) {
    document.getElementById('story').innerHTML = "You're an experienced explorer!";
} else if (playerLevel >= 1) {
    document.getElementById('story').innerHTML = "You're a brave beginner!";
} else {
    document.getElementById('story').innerHTML = "Your journey starts now!";
}

üí° Pro Tip: Check conditions in order from highest to lowest!

Comparison Operators

// The comparison toolbox
=== // Exactly equals (ALWAYS use this!)
!== // Not exactly equals
>   // Greater than
<   // Less than
>=  // Greater than or equal
<=  // Less than or equal

// Examples
let gold = 50;
let swordPrice = 30;

if (gold >= swordPrice) {
    document.getElementById('story').innerHTML = "You can afford the sword!";
}

let userInput = "secret123";
let password = "secret123";

if (userInput === password) {
    document.getElementById('story').innerHTML = "Access granted!";
}

‚ö†Ô∏è Avoid == (Use === Instead!)

‚ùå Dangerous ==

// Confusing behavior!
5 == "5"     // true (converts types)
0 == false   // true (confusing!)
"" == false  // true (what?!)

‚úÖ Safe ===

// Predictable behavior!
5 === "5"     // false (different types)
0 === false   // false (different types)
"" === false  // false (different types)

Rule: Always use === and !==

AND Operator (&&)

üîó Both conditions must be true

// Need both sword AND shield
let hasSword = true;
let hasShield = true;

if (hasSword && hasShield) {
    document.getElementById('story').innerHTML = "You're fully equipped for battle!";
}

// Multiple health and level checks
let playerHealth = 100;
let playerLevel = 7;
let hasKey = true;

if (playerHealth > 0 && playerLevel >= 5 && hasKey) {
    document.getElementById('story').innerHTML = "You can enter the dragon's lair!";
}

Think: "I need this AND this AND this"

OR Operator (||)

‚ö° At least one condition must be true

// Either magic key OR lockpicks work
let hasMagicKey = false;
let hasLockpicks = true;

if (hasMagicKey || hasLockpicks) {
    document.getElementById('story').innerHTML = "You can open the chest!";
}

// Class-based abilities
let playerClass = "Wizard";

if (playerClass === "Wizard" || playerClass === "Sorcerer") {
    document.getElementById('story').innerHTML = "You can cast spells!";
}

Think: "I need this OR this (or both!)"

NOT Operator (!)

üîÑ Reverses true/false

// Check if something is missing
let hasTorch = false;

if (!hasTorch) {
    document.getElementById('story').innerHTML = "It's too dark to see!";
}

// Reverse boolean values
let isDaytime = false;

if (!isDaytime) {
    document.getElementById('story').innerHTML = "Monsters come out at night!";
}

Think: "If I don't have this..." or "If this is not true..."

Building Smart Game Systems

// Check if ready to fight dragon
let hasSword = true;
let hasMagicStaff = false;
let hasShield = true;
let playerHealth = 60;

// Need weapon AND shield AND enough health
if ((hasSword || hasMagicStaff) && hasShield && playerHealth >= 50) {
    document.getElementById('story').innerHTML = "You're ready to fight the dragon!";
} else {
    // Give specific feedback about what's missing
    if (playerHealth < 50) {
        document.getElementById('story').innerHTML = "Your health is too low for this battle!";
    } else if (!hasShield) {
        document.getElementById('story').innerHTML = "You need a shield to protect yourself!";
    } else {
        document.getElementById('story').innerHTML = "You need a weapon to fight!";
    }
}
